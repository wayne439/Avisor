PilotAvisor — web app (Vite) for testing
===================================

Why: opening avisor as file:// blocks weather, NOTAMs, and many map tiles. This project serves the same HTML over http://localhost.

1) Install once
   cd avisor-native-shell
   npm install

2) Copy latest planner from Desktop (whenever you edit it)
   npm run sync:avisor
   Or set AVISOR_SRC to full path:
   set AVISOR_SRC=C:\path\to\avisor_v5.html
   npm run sync:avisor

3) Run dev server
   npm run dev
   Browser: http://localhost:5173/
   Click "Open PilotAvisor planner" → full app at /avisor.html

4) Production-style test (service worker)
   npm run build
   npm run preview

5) iPhone / iPad as a web app (Safari)
   On PC: npm run dev:lan  (binds 0.0.0.0 — use your Wi‑Fi IP, e.g. http://192.168.1.44:5173/avisor.html)
   On the phone: open that URL → Share → Add to Home Screen.
   Uses public/manifest.webmanifest + service worker for offline reload of cached assets.

6) Windows desktop app (Tauri — native window)
   Install Rust once: https://rustup.rs/
   cd avisor-native-shell
   npm install
   npm run sync:avisor
   npm run desktop:dev     … dev window loading /avisor.html
   npm run desktop:build   … installer under src-tauri/target/release/bundle/

7) iPhone native shell (Capacitor — requires Mac + Xcode to build/run)
   From repo root: OPEN_AVISOR_MOBILE_APP_HERE.bat → option 2 or 3
   Or: npm run cap:sync then npm run cap:open:ios

Sharing with testers over the internet (option 3 launcher)
   Default: npm run tunnel → Tunnelmole (https://....tunnelmole.net) — open /avisor.html on that host.
   If you need localtunnel instead: npm run tunnel:lt (some browsers show "not private" for *.loca.lt).

Files
  public/avisor.html   — full planner (copied from Desktop by sync script)
  src/main.ts          — dev home + offline IndexedDB demo

Planning aid only — not for sole-source navigation.
